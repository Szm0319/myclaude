# myclaude 项目 Code Wiki

## 1. 项目概述

myclaude 是一个基于 Claude 模型的智能代理系统，旨在提供一个完整的、可扩展的代理框架，支持多种功能和工具集成。该项目将多个核心机制组合成一个可运行的代理，包括工具调度、任务管理、团队协作、技能加载等功能。

**主要功能亮点：**
- 完整的代理循环与工具调度系统
- 待办事项管理与任务跟踪
- 子代理系统用于隔离执行任务
- 技能加载机制以获取专业知识
- 上下文压缩以优化模型性能
- 团队协作与消息传递系统
- 后台任务管理
- 自主运行与任务认领能力

## 2. 项目架构

项目采用模块化设计，将不同功能组织成独立的模块，通过全局实例和工具调度系统相互协作。整体架构分为以下几个主要部分：

### 2.2 模块依赖关系

- **主程序入口** → **代理主循环**：启动代理并处理用户输入
- **代理主循环** → **工具调度系统**：执行用户请求的工具操作
- **工具调度系统** → **核心功能模块**：调用各个功能模块的方法
- **核心功能模块**之间的依赖：
  - 团队管理系统依赖于消息总线和任务管理系统
  - 任务管理系统独立运行
  - 后台任务系统独立运行
  - 技能加载系统独立运行
  - 待办事项管理系统独立运行

## 3. 主要模块职责

### 3.1 代理循环模块 (s01)

**功能**：处理用户输入，调用模型，执行工具操作，管理对话历史。

**关键组件**：
- `agent_loop()`：代理的主循环函数，处理用户输入和模型交互
- 上下文压缩：自动或手动压缩对话历史，优化模型性能
- 后台通知处理：接收和处理后台任务的通知
- 收件箱检查：定期检查领导的收件箱

### 3.2 工具调度模块 (s02)

**功能**：管理和调度各种工具的执行。

**关键组件**：
- `TOOL_HANDLERS`：工具处理函数映射，包含各种工具的实现
- `TOOLS`：工具定义列表，包含工具的名称、描述和输入模式
- 工具执行：调用相应的处理函数执行工具操作

### 3.3 待办事项管理模块 (s03)

**功能**：管理待办事项列表，跟踪任务状态。

**关键组件**：
- `TodoManager`：待办事项管理器类
  - `update()`：更新待办事项列表
  - `render()`：渲染待办事项列表
  - `has_open_items()`：检查是否有未完成的任务

### 3.4 子代理模块 (s04)

**功能**：生成子代理执行特定任务。

**关键组件**：
- `run_subagent()`：运行子代理的函数
  - 支持不同类型的子代理（Explore、general-purpose）
  - 提供基础工具集供子代理使用

### 3.5 技能加载模块 (s05)

**功能**：加载和管理技能。

**关键组件**：
- `SkillLoader`：技能加载器类
  - `descriptions()`：获取所有技能的描述
  - `load()`：加载指定技能

### 3.6 上下文压缩模块 (s06)

**功能**：压缩对话历史，优化模型性能。

**关键组件**：
- `estimate_tokens()`：估计消息的令牌数
- `microcompact()`：压缩旧的工具结果
- `auto_compact()`：使用模型总结对话

### 3.7 任务管理模块 (s12)

**功能**：管理持久化的任务。

**关键组件**：
- `TaskManager`：任务管理器类
  - `create()`：创建任务
  - `get()`：获取任务
  - `update()`：更新任务
  - `list_all()`：列出所有任务
  - `claim()`：认领任务

### 3.8 后台任务模块 (s13)

**功能**：管理后台运行的任务。

**关键组件**：
- `BackgroundManager`：后台任务管理器类
  - `run()`：运行后台任务
  - `check()`：检查后台任务状态
  - `drain()`：清空通知队列

### 3.9 消息总线模块 (s09)

**功能**：处理团队成员间的通信。

**关键组件**：
- `MessageBus`：消息总线类
  - `send()`：发送消息
  - `read_inbox()`：读取收件箱
  - `broadcast()`：广播消息

### 3.10 团队管理模块 (s15)

**功能**：管理团队成员，处理团队协作。

**关键组件**：
- `TeammateManager`：团队成员管理器类
  - `spawn()`：生成团队成员
  - `list_all()`：列出所有成员
  - `member_names()`：获取成员名称列表
  - `_loop()`：成员主循环，处理成员的工作和空闲状态

## 4. 关键类与函数

### 4.1 核心类

#### TodoManager
**功能**：管理待办事项列表。

**关键方法**：
- `update(items)`：更新待办事项列表，验证每个待办事项的有效性
- `render()`：渲染待办事项列表为可读格式
- `has_open_items()`：检查是否有未完成的任务

#### SkillLoader
**功能**：加载和管理技能。

**关键方法**：
- `descriptions()`：获取所有技能的描述
- `load(name)`：加载指定技能的内容

#### TaskManager
**功能**：管理持久化的任务。

**关键方法**：
- `create(subject, description)`：创建新任务
- `get(tid)`：获取指定任务的详细信息
- `update(tid, status, add_blocked_by, add_blocks)`：更新任务状态和依赖关系
- `list_all()`：列出所有任务
- `claim(tid, owner)`：认领任务

#### BackgroundManager
**功能**：管理后台运行的任务。

**关键方法**：
- `run(command, timeout)`：在后台运行命令
- `check(tid)`：检查后台任务状态
- `drain()`：清空通知队列，获取所有通知

#### MessageBus
**功能**：处理团队成员间的通信。

**关键方法**：
- `send(sender, to, content, msg_type, extra)`：发送消息
- `read_inbox(name)`：读取指定成员的收件箱
- `broadcast(sender, content, names)`：广播消息给多个成员

#### TeammateManager
**功能**：管理团队成员，处理团队协作。

**关键方法**：
- `spawn(name, role, prompt)`：生成新的团队成员
- `list_all()`：列出所有团队成员及其状态
- `member_names()`：获取所有成员的名称列表
- `_loop(name, role, prompt)`：成员的主循环，处理工作和空闲状态

### 4.2 核心函数

#### agent_loop(messages)
**功能**：代理的主循环，处理用户输入和模型交互。

**参数**：
- `messages`：对话历史消息列表

**流程**：
1. 压缩上下文
2. 处理后台通知
3. 检查收件箱
4. 调用模型
5. 处理工具使用
6. 管理待办事项提醒

#### run_subagent(prompt, agent_type)
**功能**：运行子代理执行特定任务。

**参数**：
- `prompt`：子代理的提示
- `agent_type`：代理类型（默认为 "Explore"）

**返回值**：子代理的执行结果

#### maybe_persist_output(tool_use_id, output, trigger_chars)
**功能**：可能持久化大型工具输出。

**参数**：
- `tool_use_id`：工具使用 ID
- `output`：工具输出内容
- `trigger_chars`：触发持久化的字符数

**返回值**：原始输出或持久化标记

#### run_bash(command, tool_use_id)
**功能**：运行 bash 命令。

**参数**：
- `command`：要执行的命令
- `tool_use_id`：工具使用 ID

**返回值**：命令执行结果

#### run_read(path, tool_use_id, limit)
**功能**：读取文件内容。

**参数**：
- `path`：文件路径
- `tool_use_id`：工具使用 ID
- `limit`：行数限制

**返回值**：文件内容

#### run_write(path, content)
**功能**：写入文件。

**参数**：
- `path`：文件路径
- `content`：要写入的内容

**返回值**：操作结果

#### run_edit(path, old_text, new_text)
**功能**：编辑文件。

**参数**：
- `path`：文件路径
- `old_text`：要替换的旧文本
- `new_text`：新文本

**返回值**：操作结果

#### auto_compact(messages, focus)
**功能**：使用模型总结对话，压缩上下文。

**参数**：
- `messages`：对话历史消息列表
- `focus`：关注重点

**返回值**：压缩后的消息列表

## 6. 项目运行方式

### 6.1 环境配置

项目需要以下环境变量：

- `MODEL_ID`：使用的 Claude 模型 ID
- `ANTHROPIC_BASE_URL`（可选）：Anthropic API 的基础 URL
- `ANTHROPIC_AUTH_TOKEN`（可选）：Anthropic API 的认证令牌

### 6.2 运行步骤

1. 确保安装了所需的依赖：
   ```bash
   pip install anthropic python-dotenv
   ```

2. 设置环境变量：
   ```bash
   export MODEL_ID=your_model_id
   # 可选：export ANTHROPIC_BASE_URL=your_base_url
   # 可选：export ANTHROPIC_AUTH_TOKEN=your_auth_token
   ```

3. 运行主程序：
   ```bash
   python main.py
   ```

### 6.3 REPL 命令

程序支持以下 REPL 命令：

- `/compact`：手动压缩对话上下文
- `/tasks`：列出所有任务
- `/team`：列出所有团队成员
- `/inbox`：读取领导收件箱
- `q` 或 `exit`：退出程序

## 9. 安全考虑

- **路径安全**：使用 `safe_path()` 函数确保路径在工作目录内，防止路径遍历攻击
- **危险命令**：禁止执行危险命令，如 `rm -rf /`、`sudo` 等
- **超时控制**：对命令执行设置超时，防止无限执行
- **输出限制**：对工具输出进行限制，防止过大的输出占用过多资源

## 10. 扩展与定制

### 10.1 添加新工具

要添加新工具，需要：
1. 在 `TOOL_HANDLERS` 字典中添加工具处理函数
2. 在 `TOOLS` 列表中添加工具定义

### 10.2 添加新技能

要添加新技能，需要：
1. 在 `skills` 目录下创建新的技能子目录
2. 在子目录中创建 `SKILL.md` 文件，包含技能的元数据和内容

### 10.3 定制系统提示

可以通过修改 `SYSTEM` 变量来自定义系统提示，调整代理的行为和可用工具。

## 11. 总结

myclaude 是一个功能强大的智能代理系统，提供了完整的代理框架和丰富的功能模块。它支持工具调度、任务管理、团队协作、技能加载等多种功能，可以根据具体需求进行定制和扩展。

项目的模块化设计使得各个功能可以独立开发和测试，同时通过工具调度系统和全局实例相互协作，形成一个完整的代理系统。通过上下文压缩、后台任务管理等机制，优化了系统的性能和用户体验。

该项目可以作为开发智能代理的参考框架，也可以直接用于实际应用场景，如代码生成、数据分析、自动化任务等。
